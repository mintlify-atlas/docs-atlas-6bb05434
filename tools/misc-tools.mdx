A curated set of tools that improve Swift development workflows, from generating boilerplate code to managing Xcode projects at scale.

<Tabs>
  <Tab title="Code generation">
    Tools that automatically generate Swift source code from models, specs, or assets.

    <CardGroup cols={2}>
      <Card title="SwiftGen" icon="wand-sparkles" href="https://github.com/SwiftGen/SwiftGen">
        Auto-generates type-safe Swift code for project assets: images, colors, localizable strings, storyboards, and more. Eliminates stringly-typed resource access.
      </Card>
      <Card title="SwagGen" icon="file-code" href="https://github.com/yonaskolb/SwagGen">
        Generates a REST API client from a Swagger/OpenAPI specification using Stencil templates. Cross-platform (Linux-compatible).
      </Card>
      <Card title="Fugen" icon="figma" href="https://github.com/almazrafi/Fugen">
        Exports design resources and generates Swift code from Figma files. Keeps design tokens (colors, typography, spacing) in sync between design and code.
      </Card>
      <Card title="MemberwiseInit" icon="code" href="https://github.com/gohanlon/swift-memberwise-init-macro">
        A Swift macro (`@MemberwiseInit`) that generates memberwise initializers with the same safe-by-default semantics as Swift's built-in memberwise init, with more control over visibility and defaults.
      </Card>
      <Card title="BetterCodable" icon="layer-group" href="https://github.com/marksands/BetterCodable">
        Property wrappers that improve the ergonomics of `Codable`. Avoid writing custom `init(from:)` implementations for common patterns like date formatting, default values, and lossy arrays.
      </Card>
      <Card title="CodableWrappers" icon="brackets-curly" href="https://github.com/GottaGetSwifty/CodableWrappers">
        A collection of property wrappers that make custom serialization of `Codable` types straightforward and composable.
      </Card>
      <Card title="Model2App" icon="table" href="https://github.com/Q-Mobile/Model2App">
        Turns a Swift data model into a working CRUD app automatically. Useful for rapid prototyping.
      </Card>
      <Card title="Zolang" icon="language" href="https://github.com/Zolang/Zolang">
        A domain-specific language for generating code in multiple programming languages from a single source. Cross-platform.
      </Card>
    </CardGroup>
  </Tab>

  <Tab title="Project management">
    Tools for generating, managing, and scaling Xcode projects.

    <CardGroup cols={2}>
      <Card title="XcodeGen" icon="file-code" href="https://github.com/yonaskolb/XcodeGen">
        Generates an Xcode project file from a YAML specification and your project directory. Eliminates merge conflicts in `.xcodeproj` files and makes project structure auditable in version control.
      </Card>
      <Card title="Tuist" icon="diagram-project" href="https://github.com/tuist/tuist">
        An open-source CLI tool for creating, maintaining, and scaling Xcode projects. Provides project generation, a module graph, build caching, and team-level conventions for large codebases.
      </Card>
      <Card title="xcodeproj" icon="folder-open" href="https://github.com/tuist/xcodeproj">
        A Swift library (from the Tuist team) for reading, updating, and writing Xcode project and workspace files programmatically. Used as a foundation by Tuist and XcodeGen.
      </Card>
      <Card title="SwiftKit" icon="rocket" href="https://github.com/SvenTiigi/SwiftKit">
        A CLI tool that scaffolds a new open-source Swift framework with sensible defaults: documentation, CI, and package manifest included.
      </Card>
      <Card title="SwiftPlate" icon="copy" href="https://github.com/JohnSundell/SwiftPlate">
        Generates cross-platform Swift framework projects from the command line. Quickly produces an Xcode project that compiles for iOS, macOS, watchOS, and tvOS.
      </Card>
      <Card title="Swiftbrew" icon="beer-mug-empty" href="https://github.com/swiftbrew/Swiftbrew">
        Homebrew for Swift packages. Installs Swift package executables globally, similar to how Homebrew installs system tools.
      </Card>
    </CardGroup>
  </Tab>

  <Tab title="Xcode tools">
    Tools that improve the Xcode build and development experience.

    <CardGroup cols={2}>
      <Card title="xcbeautify" icon="paintbrush" href="https://github.com/cpisciotta/xcbeautify">
        Reformats `xcodebuild` output into a readable, human-friendly format. Particularly useful in CI pipelines where raw `xcodebuild` logs are hard to parse.

      <Card title="Toybox" icon="box-open" href="https://github.com/giginet/Toybox">
        Manages Xcode Playgrounds from the command line. Create, list, and open playgrounds without opening Xcode.
      </Card>
      <Card title="Surmagic" icon="wrench" href="https://github.com/gurhub/surmagic">
        A CLI tool that builds XCFrameworks for multiple platforms (iOS, Mac Catalyst, tvOS, macOS, watchOS) in a single command.
      </Card>
      <Card title="Swiftify" icon="arrow-right-arrow-left" href="https://swiftify.com/#/converter/code/">
        An online code converter and Xcode extension that translates Objective-C code to Swift. Useful when migrating legacy codebases.
      </Card>
    </CardGroup>
  </Tab>

  <Tab title="Code quality">

## Quick reference

<AccordionGroup>
  <Accordion title="Eliminate .xcodeproj merge conflicts">
    Use [XcodeGen](https://github.com/yonaskolb/XcodeGen) or [Tuist](https://github.com/tuist/tuist) to generate your `.xcodeproj` from source-controlled YAML or Swift files. Generated project files are never committed, so merge conflicts disappear entirely.
  </Accordion>

  <Accordion title="Speed up CI builds">
    Run [xcprofiler](https://github.com/giginet/xcprofiler) locally to find slow-to-compile expressions. Pipe `xcodebuild` output through [xcbeautify](https://github.com/cpisciotta/xcbeautify) in CI to make logs actionable.
  </Accordion>

  <Accordion title="Keep asset code type-safe">
    Add [SwiftGen](https://github.com/SwiftGen/SwiftGen) to your build process to generate enums for images, colors, strings, and storyboards. Combine with [Fugen](https://github.com/almazrafi/Fugen) if your design tokens live in Figma.
  </Accordion>

  <Accordion title="Generate API clients from OpenAPI specs">
    Use [SwagGen](https://github.com/yonaskolb/SwagGen) with a Stencil template to generate a typed Swift API client from your backend's OpenAPI specification. Re-run generation whenever the spec changes to stay in sync automatically.
  </Accordion>
</AccordionGroup>
